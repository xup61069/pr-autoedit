# PR剪教學一條龍 —— Premiere 面板(CEP)

在 Premiere Pro 裡按一顆按鈕跑完整條自動剪輯,再把剪好的序列與字幕匯回來,
完全不用離開 Premiere。

> 面板刻意做得很薄:真正的功能(降噪、轉錄、剪輯)都在上層的 Python 程式裡,
> 面板只負責啟動 Python 和匯入結果。

## 運作方式

```
面板按鈕 → 執行 ../pipeline.py 你的影片 → 匯入 output/影片名/04_project.xml + 字幕
```

## 安裝(只需做一次)

**建議做法**:回到專案根目錄雙擊 **「安裝.bat」**,它會一併把 Python、
ffmpeg、套件、面板全部裝好,路徑也自動填,不用手動設定任何東西。
以下是想自己手動裝的做法。

1. **開啟開發者模式**:雙擊執行 `enable-debug-mode.reg`(允許載入自製面板)。

2. **安裝面板**:雙擊執行 `install.bat`(把本資料夾連結到 Premiere 擴充目錄)。

3. **重啟 Premiere Pro**,然後在選單:
   `視窗 (Window) > 擴充功能 (Extensions) > PR剪教學一條龍`

> **路徑怎麼來的**:面板會自己找到專案資料夾(從自己的位置往上一層),
> Python 則優先用專案裡的 `venv`。要手動指定的話,編輯
> `config/panel.json` 的 `project_dir` / `python` 兩個欄位即可,
> 不用再改 `js/main.js`。

## 使用

1. 先在 Premiere 開啟(或新建)一個專案。
2. 面板上按「選擇影片」挑你的錄影檔。
3. (可選)展開「⚙ 設定」調參數,見下。不調也能跑,預設值就能用。
4. 按「一鍵自動剪輯」,等進度跑完(狀態列黃色=處理中、綠=完成、紅=出錯;
   下方黑框是詳細訊息,錯誤行會標紅)。
5. 完成後會自動匯入剪好的序列與字幕,並自動切到那條新序列。

> 跑的時候底下會出現「**■ 停止**」。它收掉的是整棵行程樹——Python 加上
> 它開的 ffmpeg,不會留下還在背景寫檔的孤兒行程。停掉不算失敗,不會跳
> 錯誤說明;已經辨識完的部分有快取,下次重跑會沿用,不用整支重來。

> **重跑同一支影片會「換掉」上次那條序列**,專案裡不會愈堆愈多。
> 序列名稱帶影片名(例:`我的教學 自動剪輯`),所以只會換掉這支片自己的舊序列,
> 別支影片的序列不受影響。想保留兩個版本互相比較,請改用「重算剪輯」那顆
> ——它刻意保留舊序列。

## 剪輯後工具(跑完出現,重開面板也還在)

- **開啟審閱報告**:瀏覽器打開這次剪輯的報告——哪裡被剪、哪裡被快轉、
  省了多少時間、音樂段抓在哪,一分鐘掃完。
- **重算剪輯 → 匯入新序列**:改了任何設定(停頓門檻、快轉倍率、交付方式…)
  後按這顆。不重跑辨識,幾秒重算並匯入一個**新**序列;舊序列不動,隨時可反悔。
- **幫目前序列掛人聲處理(實驗)**:把 Premiere 內建的降噪 → 參數等化器 →
  動態掛到序列的每個聲音片段(音樂段除外),之後在「效果控制」隨時調。
  片段太多會拒絕,失敗也可改走穩定做法:視窗 > 音軌混音器,
  A1 軌效果插槽依序選那三個,整軌生效、片段再多也一樣快。
- **用目前序列產生字幕**:在 Premiere 剪完後按這顆,字幕依時間軸目前的
  實際樣子重新對位(刪掉的內容自動消失、快轉段自動壓縮)並匯入新的 SRT。
- **清除快取(重新辨識)**:平常不需要。改設定會自動重跑對應步驟,
  只有結果異常但設定沒動時才用。

## ⚙ 設定(面板內直接調)

展開「⚙ 設定」會自動載入所有可調參數,分兩頁:

- **基本頁**(分組:辨識、剪輯、完成後):教學類型與我的額外術語、
  辨識引擎/模型/語言、交付方式(baked 直接剪好 / live 全保留+標籤)、
  停頓處理方式、快轉倍率、靜音門檻、音樂保護、雜音剪除、能量微剪。
- **進階頁**(平常不用動,每項有白話說明):降噪外掛(VST)、
  冗詞與口頭禪、字幕、審閱標記、畫面活動、音樂/音效保護、其他微調。

小技巧:數字滑條**點兩下恢復預設值**;改了辨識相關設定(引擎/模型/詞庫)
不用清快取,下次剪輯會自動重新辨識。

**改了就存,沒有儲存按鈕**——動完任何一項會自動寫進
`config\settings_local.json`(不動共用設定、更新專案不會被蓋),
存好時旁邊會顯示「已儲存 ✓」。

教學類型旁邊的「✎ 編輯類型」可以改內建詞庫、也可以自己開新類型,
存在 `config\vocab_local.json`(同樣不進版控)。編輯時會即時顯示用掉多少
額度——那個上限是 Whisper 的硬限制,超過的詞它直接看不到而且不會報錯。

> 欄位是由 `..\ui_settings.py` 自動產生的;之後專案新增參數,面板會自動長出對應控制項。

## 疑難排解

- **面板選單裡找不到**:確認做了步驟 1(reg)和 2(install.bat),並已重啟 Premiere。
- **按下去說找不到 Python**:改 `config\panel.json` 的 `python` / `project_dir`
  兩個欄位(**不是**改 `js\main.js`)。面板的錯誤訊息會直接印出它目前用的路徑,
  對照一下就知道找錯在哪。
- **想看詳細錯誤**:面板下方的黑框會顯示 Python 的完整輸出,錯誤行標紅。
- **面板改版後沒更新**:面板是用 junction 連結的,改完存檔重開面板即可;
  改 `manifest.xml` 才要重啟 Premiere。CSS 沒更新的話把 Premiere 完全關掉再開。
