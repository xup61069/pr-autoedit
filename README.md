# PR剪教學一條龍

**錄完教學片丟進去,幾分鐘後在 Premiere 裡拿到一條剪好的序列。**

自動去冗詞、剪停頓、保護示範音樂、剪掉咳嗽、上繁中字幕,
再把剪好的序列與字幕直接送回 Premiere。你只要跳著確認切點,通過就輸出。

實測一支 17 分鐘的教學片自動剪成 9 分鐘;原本兩三小時的剪輯,
變成 5~10 分鐘看過就好。

剪太兇或太保守?改個設定按「重算剪輯」,幾秒出一條新序列——**舊的不會被動到**,
兩條擺著比較,不滿意就刪。

錄影軟體把長片切成好幾個檔?**一次全選起來就好**,會自動接成一支處理——
一條序列、一份字幕、一份報告,不用自己先合併。

授權 MIT。

## 需要什麼

| | |
|---|---|
| 系統 | Windows 10 / 11 |
| 軟體 | Premiere Pro 2024 或更新 |
| 顯卡 | 建議 NVIDIA。沒有也能跑,但語音辨識會慢 5~10 倍 |
| 硬碟 | 約 10GB(語音辨識模型佔大部分) |

Python、ffmpeg 這些不用你懂,安裝程式會處理。

## 它怎麼運作

```
你的錄影檔
   ↓  抽音軌、響度標準化到 YouTube 標準
   ↓  語音轉文字(詞級時間戳 —— 整個系統的唯一真相來源)
   ↓  決策:冗詞 → 停頓 → 音樂保護 → 雜音 → 微剪 → 畫面判定
   ↓
Premiere 序列 + 繁中字幕 + 審閱報告
```

決策這一步做的判斷:

- **去冗詞**——「嗯、呃」看到就刪;「就是、然後」看語境,沒把握的標記起來讓你確認。
- **停頓處理**——三選一。預設是**看畫面決定**:畫面在動(你在默默示範操作)就快轉帶過,
  畫面靜止才剪掉。只聽聲音的話,那幾秒示範會整段消失而且你不容易發現。
- **音樂保護**——沒講話但有聲音的段落(示範音樂、音效)原樣保留,不會被當停頓剪掉。
- **雜音剪除**——咳嗽、清喉嚨、滑鼠聲直接剪掉;夠長的才當示範音樂保護。
- **能量微剪**——連句子中間的小空檔也剪。17 分鐘的片再省 2~3 分鐘,不損失內容。

## 安裝

### 沒碰過程式:用安裝版

到 [Releases](https://github.com/xup61069/pr-autoedit/releases) 下載壓縮檔,
解開到**沒有中文和空格**的路徑(例如 `C:\pr-autoedit`),雙擊 **安裝.bat**,
重開 Premiere,選單 `視窗 > 擴充功能 > PR剪教學一條龍`。

安裝程式會自動處理 Python、ffmpeg、顯卡加速版的辨識引擎和面板,
中間不用你做任何判斷。手把手版本看 **[新手指南.md](新手指南.md)**。

### 會用命令列

```
python -m venv venv && venv\Scripts\activate
pip install torch --index-url https://download.pytorch.org/whl/cu128
pip install -r requirements.txt
python pipeline.py 你的影片.mp4
```

> CUDA 版本要對得上顯卡:RTX 50 系列用 `cu128`,30/20 系列用 `cu121`。
> **裝錯會裝得起來但跑不動**——辨識時才失敗,不是安裝時。
> 細節見 **[SETUP.md](SETUP.md)**。

## 你會拿到什麼

跑完後,`output/影片名/` 底下只有四個給人看的檔(中繼檔收在 `_work/`):

| 檔案 | 做什麼 |
|------|--------|
| `04_report.html` | **先開這個。**剪了哪裡、省多少時間、哪些切點沒把握,一分鐘掃完 |
| `04_project.xml` | 剪好的 Premiere 專案(用面板的話會自動匯入) |
| `04_subtitles.srt` | 繁中字幕,標點感知斷行、英文術語不被切斷 |
| `01_clean_av.mp4` | 響度標準化後的影片,序列引用的就是它 |

## 面板才是主要用法

命令列只是給想接自動化的人。平常請用 `premiere-panel/` 這個 Premiere 面板:
**選影片、調設定、一鍵剪輯、匯入、重算全部在 Premiere 裡完成**,
安裝版已經幫你裝好。

面板還有這些:

- **停止**——跑到一半想中斷就按,連 Python 底下開的 ffmpeg 一起收乾淨。
- **重算剪輯**——改完設定幾秒出新序列,舊的保留著讓你比較。
- **掛人聲處理**——把 Premiere 內建的降噪 → 參數等化器 → 動態掛上去。
- **用目前序列產生字幕**——你自己又剪過之後,字幕重新對準時間軸。
- **設定組合**——「激進」「保守」一鍵切換,也可以存自己的。

詳見 **[premiere-panel/README.md](premiere-panel/README.md)**。

## 剪得不滿意?

改設定按「重算剪輯」就好,**不用重跑辨識**,幾秒出新序列。所有參數都能在面板調
(有白話說明和滑條),或直接改 [`config/settings.py`](config/settings.py)。

最有效的幾個:

| 設定 | 做什麼 |
|------|--------|
| **教學類型** + **我的額外術語** | 改善辨識最有效的一招。內建十類(motion / PV / 剪輯 / 編曲 / 特效 / 3D / 動畫 / 遊戲 / 程式 / 攝影),面板可以自己編輯或開新類型 |
| **停頓處理方式** | 看畫面決定(建議)/ 一律快轉 / 一律剪掉 |
| **靜音判定門檻** | 停頓多久才算停頓。講話慢的調高 |
| **能量微剪:保留緩衝** | 剪完覺得太急促就調大 |
| **剪掉短促雜音** | 咳嗽、滑鼠聲要不要剪 |

> **個人設定不會被更新蓋掉**:面板存的在 `config/settings_local.json`、
> 自訂詞庫在 `vocab_local.json`、設定組合在 `presets_local.json`,
> 三個都不進版控。

## 為什麼做這個

在 AI 內容氾濫的時代,我更希望看到「活人」繼續出來分享知識——有溫度、有觀點、
真實踩過坑。而且我相信,這樣的內容大家還是愛看。

但做教學影片最累的往往不是「講」而是「剪」:降雜訊、砍冗詞、剪停頓、上字幕,
一支片能耗掉兩三個小時。很多人因此懶得做,或做幾支就累到放棄。

想法很簡單:

> **讓 AI 去做無聊的剪輯苦工,把時間還給創作者,讓人專心把知識講好。**

AI 在這裡是助手,不是取代者——所以它**不直接輸出成品**,而是剪好一個
「待你確認」的版本,最後拍板的永遠是你這個活人。

## 設計重點

給想改它、或想知道它為什麼這樣做的人:

- **只轉錄一次。**詞級時間戳是唯一真相來源,冗詞、停頓、字幕全從它衍生,
  不會出現「字幕跟畫面對不上」這種各算各的問題。
- **共用單一映射表。**字幕和 Premiere 標記用同一份重映射,保證永遠對齊。
  這是整個系統的地基,`core/remap.py` 只有這一份實作。
- **信心分級。**必刪的不下標記;沒把握的才下——審閱時只要看那些就好。
- **快取分層。**調剪輯門檻只重算決策、幾秒完成;改了辨識設定(引擎/模型/詞庫)
  會自動偵測並重新辨識,不會拿舊結果充數。
- **隨時可反悔。**「重算剪輯」永遠產生新序列;降噪掛在 Premiere 當效果,
  不烘死在音檔裡。
- **審閱優先。**不直接輸出成品,而是產出剪好的專案讓你把關,誤判可以直接改。
- **面板做得很薄。**真正的邏輯全在 Python,面板只負責啟動和匯入——
  日後要換掉 UI 技術,影響面很小。

### 兩種交付方式

- **baked**(預設):匯入就是剪完的成品,搭配「重算剪輯」隨時調。
- **live**(活專案):所有片段保留、只切開並上顏色標籤(粉紅=靜音、青綠=音樂、
  紫=冗詞),進 Premiere 用「標籤 > 選取標籤群組」自己批次處理。
  片段數量多,長片會讓 Premiere 變卡,所以是備選。
